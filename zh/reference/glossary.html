Agent Harness 术语表

代理架构工作空间的规范术语。这些定义带有倾向性——旨在消除更广泛领域中的歧义。

核心循环

代理运行框架 (Agent Harness)
管理基于 LLM 的代理执行的外围基础设施：包括循环、工具系统、上下文管理、权限和会话生命周期。去掉 LLM、工具和 UI 之后剩下的部分就是运行框架。
避免使用: agent framework (过于宽泛), agent runtime (过于宽泛)。
核心循环 (Core Loop)
基本的执行循环：接收输入 → 调用 LLM → 执行工具 → 将结果反馈回去 → 重复或停止。也称为"思考-行动-观察循环"或"ReAct 循环"。
避免使用: agent loop (与 message pump 存在歧义)。
回合 (Turn)
核心循环的一个完整周期：一次 LLM 调用加上其所产生的所有工具调用的执行，当满足以下任一条件时结束：(a) 模型只返回文本，(b) 停止钩子触发，或 (c) 发生错误/中止。一次用户请求可能跨越多个回合。
避免使用: step, iteration (存在歧义——有些代码库用 "step" 表示单次 LLM 调用，不论是否涉及工具)。
迭代 (Iteration)
一个回合内的单次 LLM API 调用。一个有 N 轮工具执行的回合包含 N+1 次迭代（1 次初始调用 + N 次工具批次后的后续调用）。
避免使用: step (claude-code 用 "step" 表示我们所称的 "turn")。
嵌套双循环架构 (Nested Two-Loop Architecture)
一种核心循环结构，外层 while (true) 处理跟进消息，内层 while (hasToolCalls) 处理工具回合。pi-mono 和 openclaw 采用此架构。
避免使用: double loop (过于模糊)。
扁平 while-true 架构 (Flat While-True Architecture)
一种核心循环结构，使用单个 while (true)，其中所有关注点——上下文准备、LLM 调用、工具执行、恢复——都是单次迭代内的顺序阶段。claude-code 和 hermes-agent 采用此架构。
避免使用: single loop (存在歧义)。

工具系统

批量执行 (Batch Execution)
在 LLM 响应流完成之后，并行或顺序执行响应中的所有工具调用。实现更简单但延迟更高。
避免使用: post-stream execution。
流式执行 (Streaming Execution)
在 LLM 于流中逐个输出工具调用时就立即执行，不等完整响应返回。延迟更低但需要谨慎的状态管理。
避免使用: eager execution (不够具体)。
工具结果终止 (Tool Result Termination)
一种工具结果标志（terminate: true），表示核心循环应在该工具批次后停止。用于工具的输出本身就是最终答案的场景。
避免使用: final tool, terminal tool。

上下文管理

上下文精简 (Compaction)
缩小对话上下文的大小以保持在模型的上下文窗口内。可能采用摘要（由 LLM 生成摘要）、截断（丢弃最旧的消息）或微精简（将单个工具结果替换为简短摘要）的方式。
避免使用: compression (暗示无损，而精简很少是无损的)。
主动精简 (Proactive Compaction)
在溢出发生之前精简上下文，通常在接近令牌阈值时触发。避免浪费会因 413 错误而失败的 API 调用。
避免使用: pre-emptive compaction。
被动精简 (Reactive Compaction)
在收到 413（上下文过大）错误后精简上下文。实现更简单但会浪费一次失败的 API 调用。
避免使用: emergency compaction。

生命周期

会话 (Session)
用户与代理之间持久的、可寻址的对话。具有 ID，在代理重启后依然存在，并累积消息历史。可以跨越多次用户请求。
避免使用: conversation (过于通用)。
引导 (Steering)
在运行过程中注入的一条用户消息，用于中断并重定向代理。代理完成当前工具执行后，在下次 LLM 调用之前处理引导消息。
避免使用: interruption (引导是一种带有具体内容的中断)。
跟进消息 (Follow-up)
在代理运行期间排队的一条消息，应在代理即将停止之后处理。用于子代理结果、通知命令和多步骤编排。
避免使用: deferred message, queued message。
