第10课：Checkpoint、中断/恢复与流式传输

LangGraph 如何在超级步之间持久化状态、暂停等待人工输入，并以每种粒度流式传输执行过程。

课程二：Agent 基础设施 · 2026年6月 · 永久链接

第9课介绍了 Pregel 执行模型——LangGraph 如何用图超级步替代 while 循环。本课涵盖运行时在超级步之间做了什么：持久化状态、暂停等待人工输入、发出流事件。这三项能力使 LangGraph 超越了图执行器——使其成为生产级 agent 运行时。

Checkpoint：每一步的状态持久化

没有 checkpoint，崩溃的 agent 丢失一切。有了 checkpoint，执行可以从上次停止的地方恢复，从任意点分支，或回到之前的状态。LangGraph 的 checkpoint 系统定义于 checkpoint/base/__init__.py。Checkpoint TypedDict（第 92 行）包含 channel_values（channel 的当前值）、channel_versions（每个 channel 的版本号）和 versions_seen（每个节点消费了哪个版本，这是确定性重放的关键）。

BaseCheckpointSaver（第 176 行）定义持久化接口，有四种方法：get_tuple(config) 获取 checkpoint、put(...) 存储、put_writes(...) 存储中间写入、list(config) 列举。Checkpoint 在执行生命周期的三个点写入：输入 checkpoint（source="input"）、循环 checkpoint（source="loop"，每次 superstep 后）、以及 fork checkpoint（source="fork"，从历史分支时）。

InMemorySaver
基于 Dict，仅开发用。进程重启后不持久化。
SqliteSaver
单文件数据库。适合本地生产使用。
PostgresSaver
多进程，高可用。生产规模。

DeltaChannel 将 checkpoint 存储从 O(N²) 优化到 O(N)，每 50 步快照一次，通过回溯祖先并重放增量写入来重建中间状态。这正是 deepagents 在 graph.py:67 中使用的机制。

interrupt()：暂停按钮

interrupt(value) 位于 langgraph/types.py:811：抛出 GraphInterrupt → 持久化 checkpoint → 发出中断事件 → 客户端调用 Command(resume=value) 恢复。Interrupt 数据类（第 533 行）携带 value 和用于定向恢复的 id。这是两个课程一模式的基础设施：deepagents 的 HumanInTheLoopMiddleware（使用 interrupt_on 配置控制工具调用）和 deer-flow 的 ClarificationMiddleware（使用 Command(goto=END) 拦截澄清请求）。

8 种流模式

模式输出内容用途
values每次 superstep 后的完整状态同步整个 UI 状态
updates仅变更的 channel高效增量同步
messages逐 token 的 LLM 输出聊天式流 UI
custom任意 StreamWriter 事件进度条、通知
checkpoints保存后的 checkpoint 元数据调试状态转换
tasks任务生命周期事件监控并行执行
debug所有内部事件开发和追踪
tools工具调用事件流式传输工具执行进度

v3 流协议 GraphRunStream 位于 stream/run_stream.py:31，使用 StreamMux 将事件多路复用到多个投影。deer-flow 的 worker.py:312 使用 agent.astream(input, stream_mode=["values", "messages"])——values 模式用于完整状态快照，messages 模式用于增量文本。StreamBridge 将每个 LangGraph 流事件转换为前端的 SSE 事件。
