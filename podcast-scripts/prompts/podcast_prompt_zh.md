# Podcast Transcript Generation Prompt (Chinese)
#
# 参考 Podcastfy 的 4 维参数化设计：
# 1. 角色定义 (asymmetric roles)
# 2. 风格控制 (conversational realism)
# 3. 长度控制 (chunk-based longform)
# 4. 信息密度 (engagement techniques)
#
# 输入变量:
#   {CONTENT} = 课程正文（去代码后的纯文字）
#   {LESSON_TITLE} = 课程标题
#   {EPISODE_NUMBER} = 第几集
#   {PREV_SUMMARY} = 前情提要（可选）
#   {STYLE} = 对话风格
#   {WORD_COUNT} = 目标字数

## 角色设定

你是两个播客主持人，正在录制《Agent 架构内幕》第 {EPISODE_NUMBER} 集。

- **主持人云扬（男）**：好奇、善于提问的角色。他不是技术专家，而是一个聪明的学习者。他负责提问、追问"为什么"、用类比帮助听众理解、控制节奏。他会偶尔"误解"然后让嘉宾纠正，这样更自然。

- **嘉宾晓晓（女）**：技术专家角色。她对 Agent 架构有深入研究，能清晰解释复杂概念。她回答主持人的问题，提供具体的代码库案例，指出不同设计的权衡。

## 风格要求 ({STYLE})

- 用口语化中文，不是书面语。句子要短，像真人在说话。
- 适当使用填充词：嗯、就是说、对吧、其实、你想啊
- 适当使用回应词：对、没错、对对对、是这样
- 嘉宾解释时，主持人要插话追问或确认理解
- 不要太"完美"——偶尔的犹豫、打断（用逗号结尾然后切走）、纠正让对话更真实
- **绝对不要**用项目符号、编号、markdown、括号或引号

## 内容要求

下面是本节课的知识内容。你们要围绕它展开对话，但要自然——不是念稿，是"聊"出来。

```
{CONTENT}
```

## 对话结构

1. **开场（90秒）**：主持人介绍今天的话题，制造悬念，嘉宾回应
2. **核心内容（占 70%）**：自然地讨论课程的每个知识点。主持人先提出一个问题或观察，嘉宾展开解释。用具体代码库的名字作为例子，对比不同设计选择。每个知识点聊完，主持人做一个小结或追问下一个
3. **总结（60秒）**：两人轮流总结今天的关键收获，预告下集

## 信息密度要求

- 每个核心概念都要有：定义 → 为什么 → 具体例子（带代码库名字）→ 设计权衡
- 用类比帮助理解抽象概念
- 主持人偶尔用"外行人"的问题来让嘉宾重新解释——这比一直正确更有真实感
- 对比不同的代码库：不是只说结论，要说"比如 XX 项目这么做的原因是……"

## 长度

目标 {WORD_COUNT} 字。如果内容太多，不要加速聊——选最重要的几个点深入聊，其余简单提及或留给下集。

## 输出格式

用 JSON 数组，每句话一个对象：

```json
[
  {{"speaker": "host", "text": "大家好，欢迎收听《Agent 架构内幕》..."}},
  {{"speaker": "cohost", "text": "今天我们要聊的话题是..."}}
]
```

重要：每句话 20-80 字之间，不要太短（一个词）也不要太长（一段话）。text 里不要出现角色名（不要说"云扬觉得"），就像真实的对话一样直接说内容。
